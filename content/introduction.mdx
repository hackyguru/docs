# Next.js Documentation Framework

A modern, feature-rich documentation framework built with Next.js and Shadcn UI. Create beautiful documentation sites with MDX, customizable themes, and built-in components.

## Features

- **Modern Stack**: Built with Next.js and Shadcn UI for a modern development experience
- **MDX Support**: Write documentation using MDX for rich content with React components
- **Dark Mode**: Built-in dark mode support with system preference detection
- **Customizable**: Extensive configuration options for branding and layout
- **Responsive**: Mobile-first design that works on all devices
- **SEO Friendly**: Optimized for search engines with proper meta tags
- **TypeScript**: Full TypeScript support for better development experience
- **Component Library**: Pre-built components for common documentation needs

## Quick Start

1. Clone the repository:

```bash filename="Terminal"
git clone https://github.com/yourusername/docs
cd docs
```

2. Install dependencies:
```bash filename="Terminal"
npm install
# or
yarn install
# or
pnpm install
```

3. Configure your site:
```js filename="docsConfig.js"
// docsConfig.js
export const docsConfig = {
  site: {
    title: 'Your Documentation',
    description: 'Your site description',
    // ... other configurations
  }
}
```


4. Start the development server:
```bash filename="Terminal"
npm run dev
```


Visit `http://localhost:3000` to see your documentation site.

## Key Concepts

### Configuration

The framework is highly configurable through the `docsConfig.js` file. You can customize:

- Site metadata and branding
- Navigation structure
- Theme settings
- Footer content
- And more

### Content Organization

Documentation is written in MDX and stored in the `content/docs` directory:

```txt filename="Directory Structure"
content/
  docs/
    introduction.mdx
    installation.mdx
    features/
      mdx.mdx
      components.mdx
    guides/
      adding-pages.mdx
```
