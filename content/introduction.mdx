# Introduction

Welcome to the documentation framework built with Next.js and Shadcn UI. This modern documentation site generator helps you create beautiful, fast, and accessible documentation sites with minimal setup.

## Features

- 📝 **MDX Support** - Write content in MDX with full support for React components
- 🎨 **Beautiful UI** - Built with Shadcn UI components for a modern look
- 🌙 **Dark Mode** - Automatic dark mode support with system preference detection
- 📱 **Responsive** - Fully responsive design that works on all devices
- 🔍 **On This Page** - Automatic table of contents with active section highlighting
- 🧭 **Navigation** - Customizable sidebar and top navigation
- 🔎 **Search** - Full-text search support (coming soon)
- ⚡ **Fast** - Built on Next.js for optimal performance
- ♿ **Accessible** - WCAG compliant and keyboard navigable
- 🎯 **SEO Optimized** - Built-in SEO components and meta tags

## Getting Started

To start using this documentation framework:

1. First, install the framework using npm:
```bash
npm create next-app@latest my-docs -- --typescript --tailwind
cd my-docs
```

2. Install the required dependencies:
```bash
npm install @radix-ui/react-icons lucide-react next-themes
```

3. Configure your documentation in `docsConfig.js`

4. Start adding your content in MDX files

## Project Structure

```
my-docs/
├── content/           # Documentation content (MDX files)
├── components/        # React components
├── styles/           # Global styles and Tailwind CSS
├── public/           # Static assets
└── docsConfig.js     # Site configuration
```

## Key Concepts

### Configuration

The framework is highly configurable through the `docsConfig.js` file. You can customize:

- Site metadata and branding
- Navigation structure
- Theme settings
- Footer content
- And more

### Content Organization

Documentation is written in MDX and stored in the `content/docs` directory:

```txt filename="Directory Structure"
content/
  docs/
    introduction.mdx
    installation.mdx
    features/
      mdx.mdx
      components.mdx
    guides/
      adding-pages.mdx
```
