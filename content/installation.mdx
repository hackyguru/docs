# Installation

This guide will walk you through setting up your documentation site from scratch.

## Prerequisites

Before you begin, make sure you have:

- Node.js 18.0 or later
- npm, yarn, or pnpm package manager
- Basic knowledge of React and Next.js

## Quick Start

### 1. Create a New Next.js Project

Start by creating a new Next.js project with TypeScript and Tailwind CSS:

```bash
npm create next-app@latest my-docs -- --typescript --tailwind
cd my-docs
```

### 2. Install Dependencies

Install the required dependencies:

```bash
npm install @radix-ui/react-icons lucide-react next-themes @radix-ui/react-scroll-area @radix-ui/react-separator @radix-ui/react-slot clsx tailwind-merge
```

### 3. Configure Tailwind CSS

Update your `tailwind.config.js`:

```js filename="tailwind.config.js"
/** @type {import('tailwindcss').Config} */
module.exports = {
  darkMode: ["class"],
  content: [
    './pages/**/*.{ts,tsx,js,jsx,mdx}',
    './components/**/*.{ts,tsx,js,jsx}',
    './app/**/*.{ts,tsx,js,jsx}',
    './content/**/*.{md,mdx}',
  ],
  theme: {
    extend: {
      colors: {
        border: "hsl(var(--border))",
        input: "hsl(var(--input))",
        ring: "hsl(var(--ring))",
        background: "hsl(var(--background))",
        foreground: "hsl(var(--foreground))",
        primary: {
          DEFAULT: "hsl(var(--primary))",
          foreground: "hsl(var(--primary-foreground))",
        },
        secondary: {
          DEFAULT: "hsl(var(--secondary))",
          foreground: "hsl(var(--secondary-foreground))",
        },
        destructive: {
          DEFAULT: "hsl(var(--destructive))",
          foreground: "hsl(var(--destructive-foreground))",
        },
        muted: {
          DEFAULT: "hsl(var(--muted))",
          foreground: "hsl(var(--muted-foreground))",
        },
        accent: {
          DEFAULT: "hsl(var(--accent))",
          foreground: "hsl(var(--accent-foreground))",
        },
        popover: {
          DEFAULT: "hsl(var(--popover))",
          foreground: "hsl(var(--popover-foreground))",
        },
        card: {
          DEFAULT: "hsl(var(--card))",
          foreground: "hsl(var(--card-foreground))",
        },
      },
      borderRadius: {
        lg: "var(--radius)",
        md: "calc(var(--radius) - 2px)",
        sm: "calc(var(--radius) - 4px)",
      },
    },
  },
  plugins: [require("@tailwindcss/typography")],
}
```

### 5. Add Configuration

Create `docsConfig.js` in your project root:

```js filename="docsConfig.js"
export const docsConfig = {
  site: {
    title: 'Your Documentation',
    description: 'Documentation site built with Next.js',
    favicon: '/favicon.ico',
  },
  theme: {
    accentColor: '#0070f3',
    font: {
      sans: 'Inter',
      mono: 'JetBrains Mono',
    },
  },
  // Add more configuration as needed
}
```

### 6. Start Development Server

Run the development server:

```bash
npm run dev
```

Visit `http://localhost:3000` to see your site.

## Project Structure

```txt filename="Directory Structure"
docs/
├── app/               # Next.js app directory
├── components/        # React components
├── content/          # Documentation content
│   └── docs/         # MDX files
├── lib/              # Utility functions
└── public/           # Static assets
```

## Adding Content

1. Create new `.mdx` files in the `content/docs` directory
2. Add frontmatter with metadata
3. Write your content using Markdown and MDX
4. Update `docsConfig.js` to include new pages in the sidebar

## Configuration Options

### Site Metadata
```js filename="docsConfig.js"
{
  title: 'Your Site Title',
  description: 'Your site description',
  logo: {
    light: '/logo-light.svg',
    dark: '/logo-dark.svg',
  }
}
```

### Navigation Links
```js filename="docsConfig.js"
{
  navigation: {
    links: [
      {
        title: 'GitHub',
        href: 'https://github.com',
        external: true
      }
    ]
  }
}
```

### Sidebar Structure
```js filename="docsConfig.js"
{
  sidebar: {
    groups: [
      {
        title: 'Getting Started',
        items: [
          {
            title: 'Introduction',
            href: '/docs/introduction'
          }
        ]
      }
    ]
  }
}
```

## Customizing Styles

The documentation site uses Tailwind CSS for styling. You can customize the look and feel by:

1. Modifying the Tailwind configuration
2. Adding custom CSS in `styles/globals.css`
3. Using Tailwind classes in your MDX content
