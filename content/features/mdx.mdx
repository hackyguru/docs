# MDX Support

MDX allows you to use JSX in your markdown content. This means you can import and use React components directly in your documentation.

## Basic Usage

Create `.mdx` files in your `content` directory:

```mdx
---
title: My Page
description: A description of my page
tags: ["example", "mdx"]
---

# Welcome to My Page

This is a paragraph with **bold** and *italic* text.

## Features

- List item 1
- List item 2
- List item 3

## Code Example

\`\`\`jsx
function Example() {
  return <div>Hello World</div>
}
\`\`\`
```

## Frontmatter

Each MDX file supports frontmatter for metadata:

```yaml
---
title: Page Title
description: Page description
tags: ["tag1", "tag2"]
image: "/path/to/image.png"    # Optional image
youtube: "youtube-url"         # Optional YouTube video
---
```

## Using Components

Import and use React components in your MDX:

```mdx
import { Button } from '../components/ui/button'

# My Page

Click the button below:

<Button>Click Me</Button>
```

## Built-in Components

The framework provides several built-in components:

### Alert Component

```mdx
<Alert>
  This is an important message.
</Alert>

<Alert variant="warning">
  Warning: This is a warning message.
</Alert>
```

### Code Block

````mdx
```jsx title="example.jsx" {1,3-4}
function Example() {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  )
}
```
````

Features:
- Syntax highlighting
- Line numbers
- Line highlighting
- Copy button
- Language tabs
- File names

### Tabs

```mdx
<Tabs>
  <Tab title="npm">
    ```bash
    npm install package-name
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn add package-name
    ```
  </Tab>
</Tabs>
```

### Cards

```mdx
<Card>
  ### Feature Title
  This is a feature description
</Card>
```

## Custom Components

Create custom components in your `components` directory:

```jsx
// components/CustomComponent.jsx
export function CustomComponent({ children }) {
  return (
    <div className="custom-style">
      {children}
    </div>
  )
}
```

Use them in your MDX:

```mdx
import { CustomComponent } from '../components/CustomComponent'

<CustomComponent>
  Custom content here
</CustomComponent>
```

## Code Highlighting

The framework supports syntax highlighting for many languages:

```python
def hello_world():
    print("Hello, World!")
```

```css
.button {
    background: #0070f3;
    color: white;
}
```

```html
<div class="container">
    <h1>Hello World</h1>
</div>
```

## Advanced Features

### Table of Contents

The framework automatically generates a table of contents from your headings:

```mdx
## Section 1
Content for section 1

### Subsection 1.1
Content for subsection 1.1

## Section 2
Content for section 2
```

### Links and Images

```mdx
[Internal Link](/other-page)
[External Link](https://example.com)

![Alt Text](/path/to/image.png)
```

### Embedding Media

```mdx
---
youtube: "https://youtube.com/watch?v=..."
---

# Page with Video

The video will be displayed at the top of the page.
```

## Best Practices

1. **Organization**: Keep related content in the same directory
2. **Frontmatter**: Always include title and description
3. **Headings**: Use proper heading hierarchy (h1 > h2 > h3)
4. **Components**: Keep components simple and focused
5. **Images**: Optimize images before adding them
6. **Code**: Use appropriate language tags for syntax highlighting
